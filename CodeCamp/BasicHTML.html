Basic HTML Elements

This page contains lessons from FreeCodeCamp's Basic HTML unit in the Web Design certificate curriculum.

Basic html elements include:

  • headers
  • paragraphs
  • lists
  • images
  • links
  • forms

Headers

Use the h element to create headers. There are six header elements. The browser will render these elements in decreasing font size from h1 to h6.

Paragraphs

The p element is the basic element for text. Following the practice of typesetters, web designers use "lorem ipsum" (randomly scraped from a famous passage by Cicero) to create dummy paragraphs with placeholder text.

Lists

Lists can be ordered or un-ordered (bullets). Wrap each item in li (list item) tags. Wrap un-ordered lists in ul (un-ordered list) tags, as in the example below:

Images

You can add images to your web page by using the img element. Use the src attribute to fetch them image file. The src value can be a url or it can be a path to a local file. Links

You can use links to jump from your page to an external source or to jump to a bookmark within the same document. Use the anchor element (a) to create links. Use the href attribute to specify the url or bookmark to jump to.

Forms

Forms can be used for many purposes, including gathering data from users. For illustration purposes, the function of this very simple form can be to vote for a favorite band. There's a lot to learn about how to create and structure forms, including how to create form elements such as input fields and buttons and how to require a field to be filled.

Create a Form

Use the action attribute with the form element to create a form inviting the user to submit data. This form collects the url of a YouTube video of the band. You can nest within the form element an input element with the type attribut value set to the value "text" and a placeholder attribute set to the value "bandname" text field where the user will enter the band's name.

Create Buttons

You can also next within the form element a button element with the type attribute set to a "submit" value. You can also add the "required" attribute to the input element to require the user to enter text in the field before submitting the form.

Input Element

You can create groups of radio or checkbox buttons to collect information in a more directed manner than a text input field by setting the input attribute "type" to "radio" or "checkbox." You can create a group of buttons so the user makes one or more selections. For example, you could create a group of buttons so the user can select from among one or more groups of band traits. Let's designate two groups, "coolness" and "wanna dance." For every button in the "coolness" group, you would set the value of the name attribute to "coolness." You would then choose different indicators of coolness; let's say: hangdog, okay, most. Now set the id attribute accordingly, one button for each indicator. Finally, you will have to wrap the input elements in label elements to label each button. Best practices dictates adding the for attribute to the label element set to the same value as the input id attribute's value. So much more to learn about the input element. HTML5

Remember well-structured code? The idea of hierarchy extends to the overall structure of an HTML document. For purposes of Accessibility and Search Engine Optimization (SEO), the HTML5 adds descriptive tags. Already we had body and div to indicate sections of a document. HTML5 adds main, aside, article, and section elements to indicate sections. This document, for example, is divided into three sections that correspond to the h2 headings of Basic HTML Conventions, Basic HTML Elements, and HTML5. Header, footer, and nav indicate sections associated with particular locations in the document, while video indicates a particular type of content. More on that later in the Accessibility lessons.
