Basic CSS Conventions

CSS (Cascading Style Sheet) is a standard for styling HTML to control presentation. Important concepts are:

  • cascading
  • declarations
  • bulk application

Cascading

The word "cascading" refers to inheritancy. Style applied at a more granular level over-rides style applied at a global level. Applying style globally is more efficent; however, so best practice is to create and link an external stylesheet. Next best is to apply styles throughout the document using the style element in the document head. Finally, you can use inline styling to apply style on a case-by-case basis. The browser renders progressively, so subsequent code over-rides previous code. See this list item to which inline style has been applied to change the color.

Color

Two frequently used attributes are color which defines foreground elements and background-color.For full discussion of color styling, see MDN.

Both can be applied to any number of elements and selectors. There are several naming conventions for colors:

  1. keyword
  2. hex
  3. RGB
  4. hsla

Keyword

You can use keywords, but beware of issues documented here MDN. Here are examples:

  • tomato
  • cadet blue
  • orchid
  • sienna

NOTE: Each of these list items was styled using inline-syling, usually considered a poor practice.

RGB

This is another standard based on RGB also called the "hex triplet," since it concatenates three bytes, each representing a number in the range 00 to FF. Thus RGB (100, 100, 100) for white is #FFFFFF in hex notation.
